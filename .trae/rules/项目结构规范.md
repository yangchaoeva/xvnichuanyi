# Project: virtual-try-on (Core Context)

## 1. 核心架构与协议
- **Auth**: Cookie-based session (`auth_session`). **禁止使用 JWT**。
- **Flow**: Upload (Blob) -> Create Task (Postgres) -> Async Gen (Ark) -> Polling (Status API).
- **Stack**: Next.js 14 (App), Drizzle ORM, Vercel Blob, Cloudflare Turnstile.
- **Safety**: 严禁在日志或前端 Response 中暴露 `ARK_API_KEY` 等环境变量；错误信息需收敛为 Error Code。

## 2. 关键业务定义
- **Task Status**: `pending` | `processing` | `success` | `failed` (单向流转)。
- **API Specs**:
  - `/api/upload`: 返回 Blob URL。
  - `/api/try-on`: 触发任务，后端注入 Ark API，返回 `taskId`。
  - `/api/status`: 根据 `taskId` 轮询，成功返回结果图 URL。

## 3. 环境约束与重要文件
- **DB**: `src/db/schema.ts` (所有字段变更需同步 `db:generate`)。
- **Auth**: `src/middleware.ts` & `src/lib/auth.ts` (鉴权中心化)。
- **AI Service**: `src/services/arkImageGeneration.ts` (处理火山引擎调用逻辑)。
- **Diagnostics**: `npm run diagnose:*` (用于环境与浏览器兼容性排查)。

## 4. 协作准则 (Rules)
- **Schema First**: 修改业务逻辑前先确认 DB Schema。
- **Server Default**: 优先 Server Components；交互逻辑封装在最小颗粒度的 `'use client'` 组件内。
- **Type Strict**: 严禁 any。异步函数必须明确定义 Promise 返回类型。
- **No Hallucination**: 环境变量必须引用自已定义的配置类，严禁硬编码。
